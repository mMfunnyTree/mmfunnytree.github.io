第一题（字符串相关）

字符串计数

时间限制：C/C++语言 1000MS；其他语言 3000MS
内存限制：C/C++语言 65536KB；其他语言 589824KB
题目描述：
给定一个仅由小写字母组成且长度不超过$ 10^6 $的字符串，将首字符移到末尾并记录所得的字符串，不断重复该操作，虽然记录了无限个字符串，但其中不同字符串的数目却是有限的，那么一共记录了多少个不同的字符串？

思路

很容易想到，由于字符串的规模很大，如果不能从子串中提取出有效信息，不妨采用最简单的散列压缩，这样把一个很长的字符串$hash$成一个$64$位无符号的整数，就可以用红黑树或者hash_table查找了。

基于64位无符号长整数，可以认为：

$ hash(str[1..n]) = (str[1] + str[2]\times base + str[3]\times base^2 + …… + str[n]\times base^{n-1}) $在对$2^{64}-1$取余后的结果。

这个值在$O(len)$的预处理操作后，可以$O(1)$求出任意子串的$hash$值。

对于此题，不妨复制一遍原串，拼接到尾部，设原串为$len$，那么只需要求出所有长度为$len$的子串的$hash$值便可，去重可以使用红黑树或者hash_table。

tips:卡常（如果此处有错误请指出，非常感谢！），经过计算，64MB的内存理论上应该可以开$ 6410241024/8$个long long类型数据，可是当我一共开了$4e6$的数组时，一直提示内容超限。如果算上红黑树最坏情况下产生$1e6$个新节点，也不会超内存才对。（如果此处有问题求教。）

复杂度分析

设字符串长度为$n$，有：
时间复杂度$O(nlogn)$或者$O(kn)$，其中红黑树带$log$，哈希表带常数$k$。
空间复杂度$O(n)$，需要开长度为$n$的辅助数组。

第二题，线段树问题（难度3星）

  • 题意：
    给一个数组h[n]，表示n个人的身高，他们一字排开。$n \lt 100000$ 。
    对于所有人，求出他右边第一个比他高的人的坐标，如果不存在比他高的人，输出0。
    也就是说$for(i,inrange( 1..n) )$ 找到 $j$ 使得 $i \lt j$ && $h[i] \lt h[j]$。

  • 思路：
    线段树裸题。

    • 对于二分的区间范围内，查询区间最大值的复杂度，最朴素的是$O(n)$的，用线段树可以优化到$O(nlogn)$。
    • 所以整体复杂度$O(nlog^2n)$。

第四题，杂题(sort + map + 二分查找)（难度2星）

  • 题意
    一个公司里有$n$个人，每个人的代码能力是$v[i]$，每个人的薪水$sal[i] = v[i] - rank(v[i] in 1..n)$，其中$rank$的意思是在公司里代码能力由低到高的排名，排名从$1$开始。
    给m个询问，问薪水等于x的人有多少个。
    $n,m\lt100000 , v[i]\lt50000000000$ 。

  • 思路：
    首先对代码能力数组排序，从而可以二分查找到每个人的排名，从而可以算出每个人的薪水。
    然后由于薪水数字很大，可以使用红黑树($map$)进行映射。
    整体复杂度$O((n+m)logn)$ 。

第五题，杂题(前缀和+贪心+二分查找）（难度4星）

  • 题意：
    有一条长度无限的道路，运动员初始速度为$v$，有$n$组啦啦队给他加油，每一组的加油区间用$[x,y]$描述。
    运动员在位置$pos$时，如果有人给他加油，他就会保持当前速度前行，否则他就会降低$1$个单位的速度。

  • *速度降低到 $0$ 的时候就不跑了**。运动员可以选择任意起跑位置，求他能跑的最长距离。
    $v \lt 1e10 , x,y \lt 1e10 ,n \lt 2e5$。

  • 思路：
    首先对所有啦啦队进行排序，然后维护一个前缀和数组$pre[i]$，表示在第 $i$个啦啦队加油的时候，前面损失了多少速度。
    然后对于每个啦啦队，计算出从这个啦啦队的位置开始跑，能跑多远。这个时候二分能跑到的最后一个啦啦队的位置便可。由于预处理出$pre[]$数组，所以在进行判断时，复杂度可以达到$O(1)$
    所以整体复杂度是$O(nlogn)$。

总结

  • 知识点：动态规划，线段树，约瑟夫环（链表），二分查找，STL容器，前缀和等等。
  • 感觉自己的动态规划能力真的很差，继续加油叭……

口胡编程题

  • 如何判断B数组是不是A数组的一个子集？
    可以先把A插入进一个set，然后依次遍历B数组中的每一个元素，如果两个数组的SIZE是$O(n)$的，整体复杂度$O(nlogn)$。

  • 如何不使用额外的内存呢？
    也可以搞啊，复杂度不变，先对A数组排序，然后遍历B数组中的每一个元素，对于这个元素在已经排好序的A数组中进行二分查找，如果找到就继续，否则说明B不是A的子集。

  • 对于一个树，如何广度优先遍历？
    给我root，用一个队列，先把root压入队列，然后一直循环做如下操作：
    如果队列不空，取队首结点，把它所有儿子结点压入队列，再弹出该结点；否则退出。
    这样遍历出来的就是它的BFS序。

  • 那如何深度优先遍历呢？
    直接写个递归就行了啊……不行就用栈模拟这个过程，跟前面那个差不多。

  • A数组是升序排列的，B数组是降序排列的，如何把它们整合到一个数组C，使得C是非降序的呢？
    这个问题最简单的方法是$O(nlogn)$直接搞的吧，既然问了那就是问$O(n)$的做法咯……
    用两个指针，分别指向A的第一个位置和B的最后一个位置，然后判断这两个位置上的数字大小，再加入到C数组中去，同时移动两个指针的位置，整体复杂度$O(n)$。

  • 有100个石头，A和B取石头，每次只能取1或者2块石头，A先取，谁取完谁获胜，问A如何才能必胜？
    首先我们分析剩下1块和2块的情况，显然这个时候A是必胜的，因为他可以一次性取完剩下的石头；而剩下3块石头时，这个状态显然必败，因为(……)。
    我们预处理出这个关系表，让A每次取石头的时候都往让对方必败的状态取，一定获胜。

  • 我让你说一下这100个石头的情况下怎么做。
    显然跟3有关，首先让A取1个，剩下99个石头，那么由于99是3的倍数，而每一轮只能取1或者2块石头，可以发现，如果B取1块，A就取2块；反正A取完后保持数字是3的倍数，A必胜。

  • map和hashmap的区别？
    map红黑树实现的，很稳定。
    hashmap要看情况，如果说数据很特殊，hashmap很可能退化成链表，所以复杂度不是很稳定。

  • 简单讲讲红黑树？
    不会。（这个题目深信服也问过了，当时也说的不会……）

  • 简单讲讲快排？
    （口胡了一阵子，放弃了，说不会。）
